Common Patterns Explained:

  • Pinfu: Four sequences and a simple wait (no points from groups or wait)
  • Tanyao: Only uses number tiles 2-8 (no terminals or honors)
  • Chinitsu: All tiles from one suit only (6 han if closed, 5 han if open)
  • Kokushi: One each of all terminals and honors plus one pair (Yakuman)
  • Suuankou: Four concealed triplets (Yakuman)

üìã RonRequirements Format Reference

Custom yakus use the RonRequirements interface. Here are the main properties:

üîß Main Properties
  • requires: Requirement[] - Array of tile/unit requirements
  • requireConcealed: boolean - Must be closed hand (menzen)
  • requireOpen: boolean - Must have open melds
  • requiredTsumo: boolean - Must be self-draw win
  • requiredLastRound: boolean - Must be last round
  • requiredFirstRound: boolean - Must be first round
  • isOr: boolean - Requirements are OR (instead of AND)
  • specialCondition: function - Custom validation logic
üß© Requirement Object Properties
  • minUnits - Minimum required units (Koutsu, Shuntsu, etc.)
  • allowedSuits - Only these suits allowed
  • allowedNumbers - Only these numbers allowed
  • requiredSuits - Must contain these suits
  • requiredNumbers - Must contain these numbers
  • filteredNumbers - These numbers not allowed
  • unitRepeat - Same unit must repeat N times
  • dynamicRequirement - Dynamic conditions (Jihai, Bakaze)

üìö Examples from Built-in Yakus

Example 1: Simple Requirements (Tanyao)
{
  "requires": [
    {
      "filteredNumbers": [1, 9],
      "allowedSuits": ["MAN", "PIN", "SOU"]
    }
  ]
}

This yaku excludes terminals (1,9) and only allows number suits.

Example 2: Unit Requirements (Toitoihou - All Triplets)
{
  "requires": [
    {
      "minUnits": {
        "Koutsu": 4,
        "Toitsu": 1
      }
    },
    {
      "minUnits": {
        "Koutsu": 2,
        "Kantsu": 2
      }
    }
  ],
  "isOr": true
}

Requires either 4 triplets + 1 pair OR 2 triplets + 2 quads.

Example 3: Concealed Requirement (Pinfu)
{
  "requires": [
    {
      "minUnits": {
        "Shuntsu": 3,
        "Toitsu": 1,
        "RyanmenTatsu": 1
      },
      "requiredSuits": ["MAN", "PIN", "SOU"]
    }
  ],
  "requireConcealed": true
}

Requires sequences and must be concealed hand.

Example 4: Unit Repetition (Iipeikou - Two Identical Sequences)
{
  "requires": [
    {
      "minUnits": { "Shuntsu": 1 },
      "unitRepeat": 2
    }
  ],
  "requireConcealed": true
}

Requires the same sequence to appear twice, must be concealed.

Example 5: Dynamic Requirements (Jihai - Seat Wind)
{
  "requires": [
    {
      "minUnits": { "Koutsu": 1 },
      "dynamicRequirement": ["Jihai"]
    }
  ]
}

Requires triplet of player's seat wind (dynamic based on game state).

Example 6: Multiple Conditions with OR (Toitoihou - All Triplets)
{
  "requires": [
    {
      "minUnits": {
        "Koutsu": 4,
        "Toitsu": 1
      }
    },
    {
      "minUnits": {
        "Koutsu": 2,
        "Kantsu": 2
      }
    }
  ],
  "isOr": true
}

Accepts either 4 triplets + pair OR 2 triplets + 2 quads.

Example 7: Special Conditions (Last Round)
{
  "requiredLastRound": true,
  "requiredTsumo": true
}

Haitei Tsumo - must be last tile draw of the game.

Example 8: Mixed Conditions (Honroutou - Mixed Terminals)
{
  "requires": [
    {
      "allowedSuits": ["MAN", "PIN", "SOU"],
      "allowedNumbers": [1, 9]
    },
    {
      "allowedSuits": ["WIND", "DRAGON"],
      "allowedNumbers": [1, 2, 3, 4]
    }
  ]
}

Allows only terminals (1,9) and honor tiles.

Step-by-Step Tutorial üìñ

Step 1: Understanding Registry Format

Custom yakus use the same RonRequirements format as built-in yakus from the registry system:

Example - No Terminals: 
{
  "requires": [
    {
      "filteredNumbers": [1, 9],
      "allowedSuits": ["MAN", "PIN", "SOU"]
    }
  ]
}

What this means: Hand cannot contain 1s or 9s (terminals), and only number suits are allowed.

Step 2: Registry Properties Explained

üéØ allowedNumbers

Only these numbers are allowed

"allowedNumbers": [2,4,6,8]
üåà requiredSuits

Hand must contain these suits

"requiredSuits": ["MAN","PIN","SOU"]
üî¢ minUnits

Minimum required tile units

"minUnits": {"Koutsu": 3}
üëë requireConcealed

Hand must be concealed (menzen)

"requireConcealed": true
üö´ filteredNumbers

These numbers are not allowed

"filteredNumbers": [1,9]
üîö requiredTsumo

Must be won by self-draw

"requiredTsumo": true

Step 3: Creating Your First Custom Yaku

  1. Go to the Custom Yaku tab
  2. Enter a name for your yaku (e.g., "No Terminals")
  3. Set the han value (1-13)
  4. Paste or type your RonRequirements definition
  5. Click "Add Custom Yaku"
  6. Test it with sample hands!
